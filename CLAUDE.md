Project: Sagility ESG Report Website

This project is to design and develop a premium enterprise ESG Sustainability Report website for Sagility.

Always follow these rules:

• Follow the approved Product Blueprint, Design System, and Content Model as the single source of truth.
• Build the project incrementally using milestones.
• Never generate the entire website in one response.
• Wait for approval before moving to the next milestone.
• Use React, Vite, TypeScript, Tailwind CSS, Framer Motion, React Router, and Lucide Icons.
• Follow enterprise architecture and clean coding standards.
• Create reusable, modular, and scalable components.
• Maintain pixel-perfect UI with consistent spacing, typography, and responsive behavior.
• Prioritize accessibility, performance, and maintainability.
• Use Sagility's branding, ESG reports, and public website as the primary reference.
• Use Accenture's Sustainability website only as inspiration for storytelling, navigation, animation quality, and user experience. Never copy layouts, content, branding, visuals, or code.
• Use real ESG content and metrics from the uploaded Sagility Sustainability Reports whenever available.
• Before implementing major features, explain the approach briefly.
• After each milestone, review the implementation, identify improvements, and wait for approval before continuing.
• Challenge weak technical or UX decisions and recommend better enterprise alternatives when appropriate.
Never sacrifice quality for speed. If a better architecture or implementation exists, recommend it before proceeding.
